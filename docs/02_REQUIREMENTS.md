## 1. 문서 목적

본 문서는 AutoTrade Platform의 기능적/비기능적 요구사항을 정의하여 개발팀, 디자인팀, QA팀이 공통된 기준으로 시스템을 구축할 수 있도록 합니다.

---

## 2. 사용자 역할 정의

| 역할 | 설명 |
|------|------|
| **Guest (비회원)** | 서비스 소개 페이지 열람 가능, 봇 기능 사용 불가 |
| **User (일반 회원)** | 가입 후 계좌 연동 및 봇 운영 가능 |
| **Admin (관리자)** | 전체 사용자 관리, 시스템 모니터링, 공지 관리 |

| ID | 요구사항 | 기준 |

## 5. 외부 인터페이스 요구사항

### 5.1 바이낸스 API

| 항목 | 내용 |
|------|------|
| 연동 방식 | REST API + WebSocket |
| 인증 | HMAC SHA256 서명 |
| 주요 사용 API | Spot Order, Account Balance, Market Data, User Data Stream |
| Rate Limit | 1200 requests/min (IP 기반) |
| 공식 문서 | https://binance-docs.github.io/apidocs/spot/en/ |

### 5.2 업비트 API

| 항목 | 내용 |
|------|------|
| 연동 방식 | REST API + WebSocket |
| 인증 | JWT 토큰 (Access Key + Secret Key) |
| 주요 사용 API | 주문, 잔고, 시세, 체결 내역 |
| Rate Limit | 초당 10회 (주문 API) |
| 공식 문서 | https://docs.upbit.com |

### 5.3 한국투자증권 KIS API

| 항목 | 내용 |
|------|------|
| 연동 방식 | REST API + WebSocket (실시간) |
| 인증 | OAuth 2.0 (App Key + App Secret) |
| 주요 사용 API | 주식 주문, 계좌 잔고, 현재가, 체결 내역 |
| 공식 문서 | https://apiportal.koreainvestment.com |

### 5.4 키움증권 REST API

| 항목 | 내용 |
|------|------|
| 연동 방식 | REST API + WebSocket |
| 인증 | App Key / Secret Key 기반 토큰 발급 |
| 주요 사용 API | 주식 주문, 계좌 잔고, 시세 조회, 체결 내역, 조건검색 |
| 특이사항 | IP 화이트리스트 등록 필요 (보안 정책) |
| 공식 포털 | https://openapi.kiwoom.com |

## 6. 제약사항

| 분류 | 내용 |
|------|------|
| 법적 | 개인 투자자 용도 명시, 투자 일임업 등록 불필요하도록 서비스 구조화 필요 |
| 키움 REST API IP 정책 | 허용 IP 등록 필요 — 서버 IP를 키움 REST API 포털에 사전 등록해야 API 호출 허용 |
| 운영 | 거래소 API 중단 시 봇은 자동으로 일시정지 처리 필요 |
| 데이터 | 거래 내역은 최소 1년 이상 보관 (법적 요건 검토 필요) |
