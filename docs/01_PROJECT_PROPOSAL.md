## 2. 배경 및 목적

### 배경

알고리즘 트레이딩(자동 매매)에 대한 수요는 개인 투자자부터 소규모 펀드까지 꾸준히 증가하고 있습니다. 그러나 현재 시장에는 다음과 같은 문제점이 존재합니다.

- **파편화된 환경:** 국내 주식(증권사 HTS/API)과 코인 거래소(업비트, 바이낸스 등)를 각각 별도 도구로 관리해야 함
- **높은 기술 진입 장벽:** 자동 매매를 직접 구현하려면 Python/개발 지식이 필요
- **검증된 봇 전략의 부재:** 개인이 전략을 직접 개발하고 테스트해야 하는 부담
- **바이낸스 Trading Bots 기능의 접근성:** 바이낸스 내 봇 기능은 바이낸스 계정이 있어야만 사용 가능하며, 국내 증권사 연동은 불가

### 목적

본 프로젝트는 **국내 증권사 및 글로벌 코인 거래소 API를 통합**하여, 누구나 쉽게 자동 매매 전략을 설정하고 운영할 수 있는 **통합 자동 매매 플랫폼**을 구축하는 것을 목적으로 합니다.

## 3. 서비스 범위 및 핵심 기능

### 3.1 핵심 기능 요약

```
1. 계좌 연동 (API Key 기반)
   ├── 국내 증권사 연동 (한국투자증권, 키움증권)
   └── 코인 거래소 연동 (업비트, 바이낸스)

2. 트레이딩 봇 관리
   ├── 봇 생성 / 설정 / 시작 / 중지 / 삭제
   ├── 5가지 봇 전략 지원 (Phase 1)
   └── 봇 실행 현황 모니터링

3. 대시보드
   ├── 전체 포트폴리오 현황
   ├── 봇별 수익/손실 현황
   └── 거래 내역 조회

4. 알림
   ├── 봇 상태 변경 알림
   ├── 목표 수익 달성 알림
   └── 손절 트리거 알림

5. 전략 마켓플레이스 (Phase 2)
   └── 우수 전략 공유 및 복제
```

### 3.2 MVP 범위 (1차 출시)

- 계좌 연동: 바이낸스 1개 거래소
- 봇 기능: Spot Grid, Position Snowball, Rebalancing Bot, Spot DCA, Spot Algo Orders
- 대시보드: 기본 포트폴리오 조회
- 알림: 이메일 알림

> ℹ️ **키움증권 REST API:** 2025년 키움증권은 기존 Windows COM(OCX) 기반의 OpenAPI+를 대체하는 **키움 REST API**를 공식 출시했습니다 (포털: https://openapi.kiwoom.com). 별도 프로그램 설치 없이 Windows·Mac·Linux 모든 OS에서 HTTP 호출로 주식 매매 및 시세 조회가 가능하며, App Key / Secret Key 기반 인증을 사용합니다. 기존 OpenAPI+와 달리 서버 환경에서 직접 연동 가능하여 브릿지 서버 없이 표준 어댑터 패턴으로 구현할 수 있습니다.

## 5. 트레이딩 봇 기능 정의

바이낸스 Trading Bots 기능을 참고하여 아래 5가지 봇 전략을 구현합니다.  
참고: [Binance Trading Bots](https://www.binance.com/en/trading-bots)

### 5.1 Spot Grid (스팟 그리드 봇)

| 항목 | 내용 |
|------|------|
| 전략 설명 | 설정한 가격 범위 내에서 일정 간격으로 매수/매도 주문을 자동 반복 |
| 적합 시장 | 횡보장 (Sideways Market) |
| 지원 거래소 | 바이낸스, 업비트 |
| 핵심 파라미터 | 상한가, 하한가, 그리드 수, 총 투자금액 |
| 수익 원리 | 낮은 가격에 사서 높은 가격에 반복 매도 → 스프레드 수익 |

### 5.2 Position Snowball (포지션 스노우볼)

| 항목 | 내용 |
|------|------|
| 전략 설명 | 가격 하락 시 추가 매수를 반복하여 평균 단가를 낮추는 전략 |
| 적합 시장 | 하락 후 반등이 기대되는 시장 |
| 지원 거래소 | 바이낸스 |
| 핵심 파라미터 | 초기 투자금, 추가 매수 비율, 최대 레이어 수, 목표 수익률 |
| 수익 원리 | 분할 매수로 평균 단가 낮춤 → 반등 시 수익 실현 |

### 5.3 Rebalancing Bot (리밸런싱 봇)

| 항목 | 내용 |
|------|------|
| 전략 설명 | 다수 코인/자산의 목표 비중을 설정하고 주기적으로 자동 리밸런싱 |
| 적합 시장 | 장기 포트폴리오 관리 |
| 지원 거래소 | 바이낸스, 업비트 |
| 핵심 파라미터 | 자산 구성, 목표 비중(%), 리밸런싱 주기 또는 편차 임계값 |
| 수익 원리 | 비중이 목표 초과한 자산 매도 + 부족한 자산 매수 반복 |

### 5.4 Spot DCA (달러 비용 평균법 봇)

| 항목 | 내용 |
|------|------|
| 전략 설명 | 일정 주기마다 정해진 금액으로 자동 매수 (Dollar-Cost Averaging) |
| 적합 시장 | 장기 투자 / 우하향 시장에서의 분할 매수 |
| 지원 거래소 | 바이낸스, 업비트, 한국투자증권, 키움증권 |
| 핵심 파라미터 | 투자 자산, 1회 투자금액, 매수 주기, 총 투자 기간 |
| 수익 원리 | 가격 변동성 완화 → 평균 매입 단가 분산 |

### 5.5 Spot Algo Orders (알고 주문)

| 항목 | 내용 |
|------|------|
| 전략 설명 | 대량 주문을 TWAP/VWAP 등의 알고리즘으로 분할 실행하여 시장 충격 최소화 |
| 적합 시장 | 대량 매매 시 |
| 지원 거래소 | 바이낸스 |
| 핵심 파라미터 | 총 주문량, 실행 시간, 알고리즘 타입 (TWAP 등) |
| 수익 원리 | 슬리피지 최소화 → 유리한 평균 체결가 확보 |

## 6. 목표 사용자 (Target User)

### 주요 타겟

| 타겟 | 특징 | 니즈 |
|------|------|------|
| **개인 투자자 (중급)** | 주식·코인 투자 경험 있음, 개발 지식 없음 | 전략 설정만으로 자동 매매 원함 |
| **개인 투자자 (고급)** | 기존에 직접 스크립트로 자동 매매 운영 | 통합 관리 UI와 안정적인 운영 인프라 원함 |
| **소규모 투자 그룹** | 공동 자금 운용 | 전략 공유 및 성과 투명한 관리 원함 |

### 페르소나 (Persona)

**페르소나 A: 직장인 코인 투자자**  
- 30대 직장인, 업비트/바이낸스 사용 경험 보유  
- 시간이 없어 직접 모니터링 불가  
- "내가 잠든 사이에도 봇이 수익을 만들어줬으면 한다"  

**페르소나 B: 주식 자동 매매 경험자**  
- 40대 개인 투자자, Python으로 키움증권 자동 매매 운영 중  
- 유지보수 부담 큼  
- "잘 만들어진 플랫폼이 있다면 구독료를 낼 의향 있음"

## 9. 프로젝트 범위 및 제약사항

### In Scope (포함)

- 웹 기반 자동 매매 플랫폼 개발
- 4개 기관 API 연동 (바이낸스, 업비트, 한국투자증권, 키움증권)
- 5가지 봇 전략 구현
- 대시보드 및 포트폴리오 관리
- 사용자 인증/계정 관리
- 알림 기능 (이메일, 추후 푸시)

### Out of Scope (제외 — Phase 1)

- 모바일 앱 (iOS/Android) — Phase 3
- 선물/마진 거래 봇 — Phase 3
- AI 기반 자동 전략 생성 — Phase 3
- 소셜 트레이딩 / 팔로우 기능 — Phase 3
- 시장 데이터 수집·백테스팅 — Phase 2에서 구현

### 제약사항

| 제약 | 내용 |
|------|------|
| 법적 규제 | 국내 자동매매는 투자 일임업 관련 법규 검토 필요 (개인 용도 제한 명확화) |
| 키움 API Rate Limit | IP 화이트리스트 등록 필요, 호출 빈도 제한 준수 |
| API 호출 제한 | 각 거래소 Rate Limit 준수 필요 |
| 보안 | API Key 암호화 저장 필수 (AES-256 이상) |
